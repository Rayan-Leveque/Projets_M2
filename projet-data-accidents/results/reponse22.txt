====================================================================

Période : 2005-2015 (Groupe 1)
Source  : /home/rayan/bench-claude/data/enriched

Méthode : pour chaque année, on construit l'ensemble des couples
(Num_Acc, num_veh) présents dans usagers_{annee}.csv puis on
parcourt vehicules_{annee}.csv en vérifiant que chaque (Num_Acc,
num_veh) côté véhicule existe bien dans cet ensemble. Tout
véhicule dont la clé n'est pas trouvée est un véhicule "sans
usager".

3. Analyse / commentaires
--------------------------------------------------------------------
19580 véhicule(s) sans aucun usager rattaché,
sur 1331465 véhicules cumulés (1.4706 %).

Interprétation : un véhicule sans usager peut paraître
anormal puisqu'un véhicule est *forcément* conduit par
quelqu'un. Le cas connu correspond aux **conducteurs en
fuite** : la procédure BAAC enregistre le véhicule heurtant
(numéro, type) parce qu'il est mentionné par les témoins ou
déduit des dégâts, mais aucun usager n'est saisi côté
victime — la fiche conducteur reste vide tant que le fuyard
n'est pas identifié. Ces lignes ne sont donc pas de simples
erreurs de saisie : elles portent une information utile (un
véhicule a été impliqué sans qu'on connaisse son
conducteur).

Aucune violation n'est dûe à un num_veh vide (0
cas) : la clé est toujours non vide côté véhicules.

Décision pour les questions suivantes : ces véhicules "sans
usager" ne seront PAS supprimés ; ils restent comptabilisés
dans les analyses sur les véhicules (Q3, Q4, Q6) mais
n'apparaîtront évidemment pas dans les analyses centrées sur
les usagers (gravité, âge, sexe). La liste exhaustive est
disponible dans reponse22_violations.csv pour qui voudrait
les isoler explicitement (étude de la fuite, p. ex.).
